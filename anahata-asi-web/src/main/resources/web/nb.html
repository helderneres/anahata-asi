NetBeans ASI Studio

Deep IDE integration for the ultimate Java coding experience.

NetBeans ASI Studio transforms the IDE into a high-fidelity ASI Container. It isn't just a plugin; it's a structural bridge that gives the model direct access to project graphs, compilation lifecycles, and the industry-standard NetBeans Refactoring engine.

Architectural Principle: Always prefer NetBeans APIs (e.g., MimeLookup, EditorKit) over direct file manipulation. This ensures that the ASI respects the IDE's internal state and transactional integrity.

The Annotator Strategy

Powered by the AnahataAnnotationProvider, the Studio provides a non-intrusive, real-time feedback loop directly on your source code. It uses a Manual Delegation Chain to preserve existing IDE decorators (like Git status or Error badges).

Project Badges

Injects the Anahata bell icon and real-time session counters (e.g., [2]) into the Projects view, showing exactly which files are currently being analyzed by which ASI sessions.

Line Annotations

The ASI "paints" feedback, suggestions, or errors directly onto specific lines in the Java editor using native Annotation objects, allowing for visual-first debugging.

Maven Lifecycle Integration

The Maven toolkit gives the ASI "hands" on your build lifecycle. It leverages the NetBeans Maven Embedder to perform synchronous goal execution and recursive dependency resolution.

  • Full Transitive Analysis: getResolvedDependencies returns the final runtime classpath, providing the model with absolute situational awareness.
  • Synchronous Build Loop: Executes clean, install, or nbm-reload and captures the log output for immediate feedback.
  • The addDependency Super-Tool: A multi-phase safety tool that verifies artifact existence before modifying the POM.

Java Code Model (Index-Aware)

The CodeModel toolkit is the model's primary sensor for the codebase. It uses high-performance NetBeans indices to explore types without manual file parsing.

The "One-Shot" Advantage

Unlike standard agents that must "grep" through folders, NetBeans ASI Studio uses FQN-Resolution. Tools like getMembersByFqn allow the model to skip discovery turns and jump straight to the source of a specific class or method.

Technical Flow: Hierarchy Exploration
// Example of recursive hierarchy analysis
JavaHierarchyNode node = codeModel.getSubtypesByFqn("java.util.List", 2);
// Returns a recursive tree of implementations known to the IDE index.

Smart Refactoring Bridge

The Refactor toolkit exposes the IDE's industry-standard refactoring engine as agentic tools. This ensures that when the ASI modifies code, all references across the entire project graph remain consistent.

Symbolic Operations
  • Rename Class/Method
  • Move to Package
  • Change Method Signature
  • Inline Constant/Variable
Structural Operations
  • Extract Interface
  • Extract Superclass
  • Pull Up / Push Down
  • Encapsulate Field
Implementation: A Safe Rename
// Extracted from Refactor.java
@AiTool("Renames a file or class. This is a 'safe' rename that updates all references...")
public String rename(String filePath, String newName) throws Exception {
    FileObject fo = getFileObject(filePath);
    RenameRefactoring refactoring = new RenameRefactoring(getLookupForFile(fo));
    refactoring.setNewName(newName);
    
    // Standard NetBeans Refactoring Lifecycle:
    // 1. Pre-check -> 2. Parameters -> 3. Prepare -> 4. Execute
    return executeRefactoring(refactoring, "Rename " + fo.getName());
}

The Development Lifecycle

For contributors building new toolkits, NetBeans ASI Studio supports a high-velocity nbmreload workflow. Changes to tools in the anahata-asi-nb module or its dependencies can be deployed into the host IDE in a single turn, enabling real-time experimentation without platform restarts.
