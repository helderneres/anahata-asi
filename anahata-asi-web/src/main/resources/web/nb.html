NetBeans ASI Studio

Deep IDE orchestration for the ultimate Java coding experience. Turning NetBeans into a high-fidelity ASI Container.

NetBeans ASI Studio is not just a plugin; it's a structural bridge that gives the model direct access to project graphs, compilation lifecycles, and the industry-standard NetBeans Refactoring engine. It follows the **IDE API Preference Principle**: always prefer MimeLookup or EditorKit over direct file manipulation.

Singularity-Ready: By integrating directly with the IDE's internal state, the ASI respects transactional integrity and project-specific configurations (like Compile on Save) that are invisible to standard file-system agents.

The Annotator Strategy

The Studio provides non-intrusive, real-time visual feedback directly on the source code via the AnahataAnnotationProvider. This is achieved through a Manual Delegation Chain that preserves existing IDE decorators (like Git status or Error badges).

Project View Badges

Injected at position 10000, our provider adds the Anahata bell icon and real-time session counters (e.g., [2]) to the Projects view, indicating active AI analysis.

Line Annotations

The ASI "paints" suggestions or errors directly onto specific lines using native Annotation objects, allowing for visual-first, agentic debugging.

NetBeans Maven Embedder

The Maven toolkit leverages the host IDE's internal Maven engine to perform synchronous build goal execution and recursive dependency resolution.

  • Full Transitive Analysis: getResolvedDependencies returns the absolute runtime classpath, providing the model with total situational awareness.
  • Synchronous Build Loop: Executes clean, install, or nbmreload and captures the log output for immediate feedback in the chat.
  • The addDependency Super-Tool: A safe, multi-phase tool that verifies artifact existence and downloads sources before modifying the POM.

Java Code Model (Index-Aware)

Powered by org.netbeans.api.java.source, the CodeModel toolkit is the model's primary sensor. It explores types via high-performance indices rather than slow manual parsing.

Technical Advantage: FQN-Resolution

Tools like getMembersByFqn allow the ASI to skip "discovery turns." If the type name is known, the model jumps straight to the source, significantly reducing turn overhead and token cost.

Implementation: Subtype Search Task
// Exposing the IDE Index as an AI Tool
public JavaHierarchyNode getSubtypesByFqn(String fqn, int maxDepth) {
    // Uses NetBeans ElementHandle and ClassIndex to build a recursive tree
    // across all open projects and their dependencies.
    return searchSubtypes(fqn, maxDepth);
}

Smart Refactoring Bridge

The Refactor toolkit exposes the IDE's industry-standard refactoring engine as agentic tools. This ensures that when the ASI modifies code, every reference in every open project is updated atomically.

The Refactoring Lifecycle
1. Pre-check

Verifies the element exists and is eligible for the operation (e.g., not a read-only JAR).

2. Parameters

Validates new names or signatures (FQN-resolution, reserved words).

3. Prepare

Discovers all references and structural impact across the entire project graph.

4. Execute

Transactional application of changes to the filesystem and the active IDE buffer.

Implementation: A Safe Rename Tool
// Extracted from Refactor.java
@AiTool("Renames a file or class. This is a 'safe' rename...")
public String rename(String filePath, String newName) throws Exception {
    FileObject fo = getFileObject(filePath);
    RenameRefactoring refactoring = new RenameRefactoring(getLookupForFile(fo));
    refactoring.setNewName(newName);
    
    // Execute through the 4-phase lifecycle and return a detailed log
    return executeRefactoring(refactoring, "Rename " + fo.getName());
}

The Development Lifecycle

Contributors building new toolkits can leverage the nbmreload goal. This triggers a synchronous reload of the Anahata plugin into the host IDE instance, allowing for a tight "Code → Build → Test" loop in a single turn without platform restarts.
