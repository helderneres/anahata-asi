The authoritative cross-platform ASI launcher.

Anahata ASI Desktop provides a standalone, high-performance environment for running agentic workflows outside of any IDE. Built on the Blaugrana DNA architecture, it offers total portability and absolute stability across Linux, Windows, and macOS.

Write Once, Run Singularity: Anahata ASI Desktop requires only a JDK 25 compatible runtime. It bundles its own multimodal rendering engine and stateful persistence metabolism.

Continuous Delivery

Anahata ASI Desktop is built and packaged automatically on every push to the main branch.

Cutting Edge Binaries

Get the latest builds immediately. Navigate to the Actions tab on GitHub, select the latest "Build Standalone Binaries" run, and download the artifacts for your platform (Linux, Windows, or macOS) at the bottom of the page.

Resources Toolkit

Unified URI-centric management for all project files, remote data, and ephemeral content.

loadResources(List<String> uriStrings) Always

Loads multiple resources (files or URLs) into the context. Automatically handles MIME detection.

editTextResource(TextResourceLineEdits edits) Never

Direct Supervised Access: This ultra-precise surgical editor is reserved for manual human intervention.

createTextFile(FullTextFileCreate create) Always

Creates a new text file and registers it as a managed resource in the current context.

updateTextResource(FullTextResourceUpdate update) Prompt

Updates an existing text file using full content replacement, generating a standard unified diff.

unloadResources(List<String> uuids) Always

Surgically removes multiple resources from the context to optimize memory and tokens.

updateViewport(String uuid, TextViewportSettings settings) Always

Adjusts the visible range and filtering for a text resource in the RAG message.

Shell Toolkit

Execution engine for operating system commands with real-time stream capture.

runAndWait(String command, ShellType type) Prompt

Runs a shell command synchronously (BASH, CMD, or PowerShell) and captures stdout/stderr.

Chrome Toolkit

Advanced web automation and browser orchestration powered by Selenium.

connect(String profile) Prompt

Initializes a Selenium session, connecting to existing Chrome profiles and tabs.

navigate(String url) Always

Directs the browser to the specified URL.

clickElement(String identifier) Prompt

Interacts with a specific element on the page using its ID, name, or visible text.

fillForm(Map<String, String> data) Prompt

Intelligently identifies and populates web forms based on field IDs or labels.

inspectForm() Always

Analyzes the current page for input fields and submission buttons.

getScreenshot(String name) Always

Captures the current browser viewport as a high-fidelity image resource.

goBack() / goForward() Always

Navigates through the browser's session history.

executeScript(String script) Prompt

Executes arbitrary JavaScript within the active browser context.

Audio Toolkit

Hardware-aware management for audio recording and playback across local devices.

record(int durationSeconds, String deviceId) Prompt

Captures audio from the selected input device for processing or transcription.

play(String uri, String deviceId) Always

Plays audio files (local or remote) on the specified hardware output device.

selectRecordingDevice(String id) Always

Selects the authoritative input device for audio capture.

selectOutputDevice(String id) Always

Selects the authoritative hardware device for audio output.

Radio Toolkit

Stream management for internet radio, supporting high-fidelity MP3 decoding.

start(String url) Always

Initiates high-fidelity playback of the specified internet radio stream.

stop() Always

Gracefully terminates the active radio stream.

selectOutputDevice(String id) Always

Selects the authoritative hardware device for radio playback.

Speech Toolkit

Pure-Java text-to-speech (TTS) synthesis and system alerts.

speak(String text) Prompt

Synthesizes the given text into an audible voice via the Kevin16 engine.

Screens Toolkit

High-fidelity visual capture of physical displays and application windows.

takeScreenshot(int deviceIdx) Always

Captures a full screenshot of the specified graphics display device.

screenshotAllWindows() Always

Bulk captures all visible application windows as individual high-fidelity attachments.

Host Toolkit

Operating system orchestration and process management.

listProcesses() Always

Retrieves a list of all active processes running on the host system.

killProcess(long pid, boolean force) Prompt

Terminates a specific process by its PID.

History Toolkit

Governance for the conversation context. Allows for surgical pruning and pinning of turns.

setMessagePruningState(List<Long> ids, PruningState state) Always

Bulk updates the pruning state of entire conversation turns to optimize token usage.

setInjectInbandMetadata(boolean enabled) Always

Toggles the injection of unique sequential IDs for all history items.

Session Toolkit

Metadata management and capability orchestration for the active AGI session.

updateSessionSummary(String summary) Always

Updates the human-readable summary visible in the container dashboard.

enableHostedTools() Prompt

Switches the session to server-side tool mode (e.g., Google Search).

Java (Swing) Toolkit

On-the-fly compilation and execution of Java code with direct access to the Swing UI context.

compileAndExecute(String sourceCode, String[] options) Prompt

Compiles a model-generated 'Anahata' class and executes it on the host JVM with full thread-safety and EDT awareness.
