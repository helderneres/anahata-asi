Anahata Swing

High-fidelity, provider-agnostic UI components for building agentic workflows in Java.

The AgiPanel

The primary container for an AGI session. It coordinates the chat interface, resource management, and real-time status monitoring in a unified, responsive layout.

The "Tiki-Taka" Rendering Engine

To maintain extreme performance in conversations with thousands of turns, Anahata Swing uses a surgical Diff-and-Update rendering engine. This hierarchical strategy ensures that only the atomic units of the UI are updated, preventing "flicker" and minimizing memory pressure.

1. Message Level

The ConversationPanel tracks AbstractMessage objects and reuses AbstractMessagePanel instances via an identity-based cache.

2. Part Level

Individual parts (Text, Blob, ToolCall) use specialized AbstractPartPanel implementations determined by the PartPanelFactory.

3. Segment Level

Markdown parts are parsed into TextSegmentDescriptor records, allowing the UI to recycle individual code blocks or UML diagrams.

Multimodal High-Fidelity Rendering

Anahata Swing transforms raw domain data into interactive, human-readable components. It doesn't just show text; it visualizes the ASI's internal state.

Syntax Highlighting

Powered by RSyntaxTextArea, the CodeBlockSegmentRenderer provides IDE-grade highlighting for hundreds of languages directly in the chat.

UML & Mermaid

The MermaidCodeBlockSegmentRenderer captures Mermaid markdown and renders it as interactive SVG/PNG diagrams within the flow.

Tool Previews

The ParameterRendererFactory generates high-fidelity previews for complex tool arguments (like file diffs or JSON structures) before user approval.

Live Media

The MediaRenderer manages the lifecycle of audio playback and image rendering, ensuring resources are disposed of when the turn is recycled.

Reactive Domain Binding

The UI is a pure reflection of the Domain state. Leveraging PropertyChangeSource and EdtPropertyChangeListener, the Swing layer remains perfectly synchronized with the Core engine without direct coupling.

EDT Safety: The SwingUtils.runInEDT pattern ensures that asynchronous events from the ASI model (like streaming text or tool status updates) are marshaled safely onto the Event Dispatch Thread.

Swing-Aware Tools

The SwingAgiTool pattern enables AI tools to interact with the UI safely. This is critical for tools that need to capture screenshots, record audio, or interact with IDE components.

public class MyTool extends SwingAgiTool {
    @AiTool("Captures a UI element")
    public byte[] capture() {
        return runInEdtAndWait(() -> {
            // This logic executes safely on the Swing EDT
            return UICapture.capture(myComponent);
        });
    }
}
