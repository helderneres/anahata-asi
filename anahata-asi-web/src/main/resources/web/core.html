Anahata Core

An Enterprise ready, Domain-Driven abstraction of the Context Window. Engineered for stability, modularity, portability, observability, security and traceability.

Mission Statement

Core provides the atomic units of context (Messages, Parts, Tools) and the context augmentation and metabolic management systems (CwGC) required to sustain live, long-running, multi-modal, cross platform, super intelligent agentic sessions without provider lock-in.

The AGI Container

The Agi class is the root of the object graph. It orchestrates the lifecycle of a single AI session, managing the persistence, state, and orchestration of all other core components via its AgiConfig.

  • AgiConfig: The DNA of the session. Defines the Metabolic Rate (Default Max Depth) for different part types: Text (108), Tools (12), and Blobs (4). It handles provider discovery and latently binds the session to the host environment.
  • Persistence: Supports high-performance serialization via Kryo. The AGI state (history, tool state, config) can be passivated to disk and activated instantly, enabling "Warm Starts" for complex coding sessions.
  • Orchestration: Synchronizes the ContextManager (Memory), ToolManager (Action), and ResourceManager (Data) into a unified, thread-safe execution context.
  • Security: Implements a "Final Gate" for tool execution, enforcing ToolPermission policies (Prompt, Approve Always, Deny Never) before any Java code is invoked.

Messages & Parts

In Anahata V2, context is not a flat string; it is a rich graph of strongly-typed objects. A conversation is a sequence of AbstractMessage turns, each containing one or more AbstractPart components.

In-Band Metadata Injection

Self-awareness is achieved by interleaving invisible metadata headers before every message and part. This allows the model to see its own Part ID, Remaining Depth, and Pruning State.

Example: Model Message Header
[x-anahata-message-id: 12 | From: models/gemini-3-flash | Device: Cloud | Time: 21:05:32 | Depth: 4]
Example: Effectively Pruned Tool Call (Semantic Ghost)
[x-anahata-part-id: 19 | Type: JavaMethodToolCall | Status: EXECUTED | Remaining Depth: 0 | pruningState: AUTO | effectivelyPruned: true | Hint: Tool: NbFiles.updateTextFile | Status: EXECUTED | Args: {path=pom.xml} | Result: Success]

By including the Pruned Hint, the model maintains a persistent memory of its actions without paying the token cost of the full content. This creates a "Semantic Ghost" that I can reason about many turns later.

Garbage Collection (CwGC)

The Context Window Garbage Collector manages the metabolic health of the session. It prevents token overflows while maintaining semantic continuity through a tiered pruning policy based on Depth (distance from current turn).

  • PINNED:
    Vital parts (source code, instructions) that stay in the context indefinitely. IMMUNE to GC.
  • AUTO:
    Default state. Parts are "Soft Pruned" when their Remaining Depth reaches 0.
  • PRUNED:
    Content explicitly recycled by model or user. Can be Unpruned until Garbage Collected.

Hard GC Trigger

A message turn becomes Garbage Collectable (permanently purged) ONLY once every individual part within that message has become effectively pruned.

Toolkits & Tools

The ToolManager is the engine of action. It uses a Java-native approach where tools are simply methods in @AiToolkit annotated classes. No manual JSON schema management is required.

  • Reflective Discovery: Methods annotated with @AiTool are automatically converted into provider-agnostic schemas.
  • Turn-Persistent State: Tools can store data across turns using the sessionAttributes map, which is safely persisted.
  • Deferred Snapshotting: The AbstractToolCall captures the request, while the AbstractToolResponse captures a historical snapshot of the execution (Executed Args, Thread Context, Result).

Context Providers

Context Providers are the sensors of the ASI. They contribute high-salience, just-in-time information to the RAG Message at the end of every turn.

  • Hierarchical Augmentation: Providers can have children, creating a structured tree of information (e.g., Project -> Structure).
  • Identity & Persona: Root providers supply the system instructions that define the ASI's persona and Barcelona-supporter status.

Reactive Events

Built on top of java.beans.PropertyChangeSupport, the Core API allows any UI framework (Swing, JavaFX) to bind directly to the domain model for zero-polling updates.

Status Events

The StatusManager fires events when the model transitions (e.g., IDLE -> THINKING).

Property Binding

UI components bind to text, status, or tokenCount to update fluently.
